The setup stage presented in this tutorial demonstrated how the Django web framework creates all the necessary directory structures and python files to start building web applications. Django uses the Python programming language and compatibility exists between the different version of Python and different versions of Django. Since different versions of Django are compatible with different versions of Python, it is recommended to develop within a virtual environment. Virtual environments enable to creation of a development environment that would include compatible versions of Django and Python. Virtual environments isolate Django projects each with its own set of package versions avoiding conflicts between projects or the operating system. Virtual environments also have additional benefits like security and organization. Linux platforms like Ubuntu come with Python included in the system. If building a web application in Django requires a specific version that is different than the one that is included with the operating system, virtual environments make it possible to install different versions of Python as needed for the specific project without considering the version included on the operating system. There are different tools available that make creating and managing virtual environments easy. The methods listed below are the different ways to setup a Django project on Ubuntu system using various tools. The basic setup stage described in this tutorial follows the five procedures listed below, depending on the tools used there are other steps that are needed and are specific to the tools selected.
1) Update Ubuntu system packages.
2) Create and activate virtual environment.
3) Install Django.
4) Create Django project.
5) Create Django Applicaiton.
