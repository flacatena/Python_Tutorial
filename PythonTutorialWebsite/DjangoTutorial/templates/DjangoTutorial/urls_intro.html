Django URL

Web requests made from a web browser typically involve the input of a URL into the web browser from a user. Django incorporates a system to handle these requests and provide the corresponding responses.

Creating the View

Django handles the requests by cross referencing the URL submitted by the client to a list of URLS known as URL patterns. Django stores a list of URL patterns in the urls.py file also known as URLconf. A component of Django named the URL dispatcher matches the URL provided by the client to the list of URLS defined in the urlpatterns list in the urls.py module. After a match is made a process known as routing takes place. The dispatcher will invoke the corresponding view that is listed in the urlpatterns that matches the supplied URL from the client.
